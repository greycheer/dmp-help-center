# Product Creation and Development

## Overview

Device development is a core process within product development. It allows developers to flexibly define product capabilities and manage product firmware.

**Navigation Path:** [Product] → [Product Development] → Click **Continue Development** for a specific product

## Product Development Workflow

```
1. Create Product → 2. Configure Functions → 3. Device Interaction → 4. Firmware Management → 5. Testing & Release
```

## Accessing Product Development

1. Navigate to **Product** → **Product Development**
2. Click **Continue Development** for the target product
3. Enter the product details page
4. Select the **Device Development** subpage to perform configurations

## Development Status

Products can be in one of the following statuses:

| Status | Description | Can Edit |
|--------|-------------|----------|
| **In Development** | Initial development phase | ✅ Yes |
| **Pre-release** | Ready for limited testing | ⚠️ Limited |
| **Released** | Production ready | ❌ No (must withdraw first) |

## Device Development Subpages

### 1. Product Functions

Configure the capabilities and features of your device:
- Select from standard function library
- Define properties, actions, and events
- Configure data types and value ranges

### 2. Device Interaction

Manage interaction capabilities via App/Web:
- Product Display (icons, names, models)
- Device Provisioning Configuration
- Device Setting Configuration (Functional Grouping)

### 3. Mass Production

Generate device credentials for production:
- Generate Five-Tuples
- Export authorization codes
- Manage production batches
