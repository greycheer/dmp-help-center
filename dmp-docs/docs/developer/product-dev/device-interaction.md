# Device Interaction Configuration

## Overview

**Device Interaction** is used to manage interaction capabilities during consumer usage via the App/Web, including:

- **Product Display**: Device models and images
- **Device Provisioning Configuration**: Network setup instructions
- **Device Setting Configuration**: Functional grouping for App settings

**Navigation Path:** [Product] → [Product Development] → Click **Continue Development** → Select **Device Interaction** subpage

## Product Display

Configure the product information exposed to consumers on the App/Web side:

| Setting | Description |
|---------|-------------|
| **Product Icon** | Display icon for the product |
| **Product Name** | Consumer-facing product name |
| **Product Model** | Model identifier shown to users |

## Functional Grouping

Functional Grouping is used to manage product settings on the App and CMS. Developers can flexibly configure tree-structured function groups to meet common configuration needs during consumer use.

### Default Groups

When a product is created, the DMP platform initializes default function groups based on the product category. Developers can modify these defaults as needed.

### Creating Groups

1. Click **New Primary Group** to create a highest-level group
2. Click groups in the tree to create sub-items (lower-level groups)
3. Each group supports editing of multilingual group names and descriptions

### Group Types

| Type | Description | Function Support |
|------|-------------|------------------|
| **Group** | Configurable settings section | ✅ Can assign functions |
| **Group Name** | System placeholder for structure | ❌ No functions, name only |

**Note:** Groups whose type is "Group Name" do not support function configuration. They exist as system placeholders to maintain the tree structure and only allow name modification and adding sub-items.

### Assigning Functions

For categories whose type is **Group**:

1. Select the group
2. Click **Add Function**
3. Select from available functions (must be consistent with functions selected under Device Development → Product Functions)
4. Configure display settings

### Function Display Configuration

For configured function items, developers can adjust:

| Setting | Options | Description |
|---------|---------|-------------|
| **Display Style** | Smart Control, Read-Only Display, Custom Display | How the function appears in App |
| **Multilingual Name** | Per-language display name | Localized function name |
| **Description** | Per-language description | Help text for users |

**Available Display Styles:**
- **Smart Control**: Interactive control with automatic UI
- **Read-Only Display**: Shows current value only
- **Custom Display**: ⚠️ Not yet available to developers

**Note:** Currently, the platform only supports App-side control of **Bool** and **Enum** type attribute functions. Other function types are display-only and show the latest device-reported values.

### App Display Effect

The App-side display generated by function group configuration includes:
- Configurable settings organized by groups
- Product basic information
- Device reboot option
- Device sharing
- Device information

**Note:** Function Groups only control the configurable settings section. Other sections (product info, reboot, sharing) are built-in App features.

## Provisioning Configuration

Configure network provisioning methods for your product. See [Network Provisioning](../device-integration/network-provisioning) for detailed configuration.

### Recommended Configurations by Product Type

#### Wi-Fi Products
Configure two provisioning methods:
- **Bluetooth** (primary)
- **Wi-Fi Configuration** (fallback)

#### 4G Products
Configure four provisioning methods:
- **Bluetooth** (primary)
- **Scan Device Body Code** (secondary)
- **ID Addition** (fallback)
- **Device Code Scanning** (fallback)

#### LAN Products
Configure three provisioning methods:
- **Scan Device Body Code** (primary)
- **LAN Scanning** (fallback)
- **ID Addition** (fallback)
